# Section 1 - Non-Technical Description

This program reads a stored list of conditional trading orders for a given exchange and account, checks current market quotes and open positions, computes each order's configured profit target and stop-loss levels relative to the order price, and then prints a human-readable summary line for each stored conditional order including asset, stop, current quote, entry price, take-profit, size, direction, and for certain brokers additional details like unrealized profit, margin used, and trade duration.

# Section 2 - Technical Analysis

The script starts by importing standard libraries and two local modules (JRRsupport and JackrabbitRelay). It defines file-system paths for a data directory and constructs a "storehouse" filename pattern used to locate saved conditional order records. A locker object is created for synchronization but is not actively used in the code paths shown.

A helper function CalculatePriceExit(order, ts, dir, price) computes a numeric exit price for either a "TakeProfit" or "StopLoss" specification found inside an order dictionary. It reads the textual value from order[ts] and interprets percent values (ending with '%'), pip values (containing 'p' or 'P') and literal numeric values. Percent values are applied relative to the given price, with the sign depending on whether the direction (dir) is 'long' or 'short'. Pip values are converted using a factor of 0.0001. For StopLoss, if the order lacks a StopLoss entry the function returns 0. The function returns the computed exit price as a float.

The main() function expects command-line arguments: exchange, account, and an asset (or search) identifier. If insufficient arguments are present, it prints an error and exits. Using the exchange, account, and cleaned asset string, it constructs the path to a storehouse file (a single filename using exchange, account, and asset). If the file exists, the file contents are read into buffer (via JRRsupport.ReadFile). If the file is missing or empty, the program prints "No conditional orders yet." and exits.

When buffer contains data, it is split on newline into entries. Each non-empty entry is parsed from JSON into a Data dictionary. The code expects Data to include a 'Framework' field (examples: 'ccxt', 'mimic', 'oanda') and an 'Order' sub-dictionary. If the Order's Exchange or Account fields do not match the requested exchange/account the entry is skipped. If a search asset was provided and the Order's Asset does not match it, the entry is skipped.

For each remaining entry the program maintains Relay objects per framework in a Relay dictionary. If a Relay for the framework does not exist yet, the code constructs one via JRR.JackrabbitRelay(framework=..., payload=json.dumps(Order), NoIdentityVerification=True). If it already exists, it calls .ProcessPayload(NewPayload=json.dumps(Order)) on it. This ensures there is a per-framework relay instance used to make subsequent queries.

The program also caches ticker data per asset in TickerData. For a new asset it calls Relay[framework].GetTicker(symbol=Relay[framework].Order['Asset']) and stores the returned ticker data (expected keys include 'Bid' and 'Ask'). The current ticker for the order is assigned to ticker.

Next the code branches by Data['Framework'] type to extract order and execution details and produce printed output lines.

- For 'ccxt' entries: it reads Data['ID'], deserializes Data['Response'] if necessary, extracts price and amount from Response['Details'], and determines side: if Resp['side'] == 'buy' it uses 'long' and uses ticker['Bid'] as the current quote, otherwise 'short' with ticker['Ask']. It computes take-profit and stop-loss using CalculatePriceExit, rounds to 8 decimals, and prints a one-line formatted summary containing id, asset, stop-loss, current price with entry price in parentheses, take-profit, amount, and direction.

- For 'mimic' entries: it reads Data['ID'], deserializes Response if needed, parses Price and Amount fields as floats. It computes a duration by parsing the Detail['DateTime'] string, truncating fractional seconds to microseconds and converting to a UTC-like timestamp, then subtracting that from the current local timestamp to get a timedelta called duration. If amount >= 0 direction is 'long' and uses ticker['Bid'], otherwise 'short' with ticker['Ask']. It computes and rounds tp/sl with CalculatePriceExit, formats textual representations of the configured TakeProfit and StopLoss, and prints a formatted line that includes id, asset, stop-loss, stop-loss text, current price with entry price in parentheses, take-profit, take-profit text, absolute amount, direction, and the duration timedelta.

- For 'oanda' entries: the code sets a flag oanda=True, reads Data['ID'] and ensures openTrades[Order['Asset']] is initialized by calling Relay[framework].GetOpenTrades(symbol=Order['Asset']) if necessary. It calls Relay[framework].GetOrderDetails(OrderID=id) to retrieve detail information for the order. If detail is None it continues to the next entry. It extracts a client trade id cid from detail[-1]['id']. It computes a duration by parsing detail[0]['time'] (expecting an ISO timestamp with fractions), truncating microseconds and comparing to current time. It searches the cached openTrades list for an entry whose 'id' matches cid; when found it reads price, currentUnits as amount, unrealizedPL as rpl, and marginUsed as mused. If not found it skips the entry. It increments counters: tlen becomes the length of openTrades[Order['Asset']], tpl accumulates rpl, and tmused accumulates mused. Direction and current ticker price choice follow the same amount>=0 logic. It computes tp/sl with CalculatePriceExit and rounds to 5 decimals. It prints a detailed formatted line showing id, asset, stop-loss, stop-loss text and absolute pip distance between current quote and stop, the current quote and entry price, take-profit and take-profit text plus pip distance, unrealized P/L, absolute amount (formatted with no decimals), margin used, a single-character direction (L or S), and the duration.

If no conditional entries were present in the file buffer the program prints a message and exits; otherwise, after processing all entries, if any unrealized P/L was accumulated (tpl != 0) it prints a summary block that shows the number of current trades (tlen) and the total PNL (tpl). If any oanda entries were present it also prints the total margin used (tmused).

Finally, the module calls main() when executed as a script. The overall behavior is: read stored conditional order JSON lines for a given exchange/account/asset, instantiate per-framework relay objects, obtain current ticker and open-trade details as needed, compute configured take-profit and stop-loss price levels with different unit conventions (percent, pips, absolute), and print formatted summaries and aggregate PNL/margin data for the matched entries.
