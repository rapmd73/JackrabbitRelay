Section 1 - Non-Technical Description

This program reads one or more log files and prints a summary of daily monetary results found in those files, followed by overall counts and averages for the period covered. For each input file it lists each date found in the file along with a label derived from the filename and the total amount for that date, then it prints totals such as the number of profit/loss lines, number of losses, average per trade, average trades per month, and average profit per month.

Section 2 - Technical Analysis

The program begins by ensuring at least one command‑line argument (a log filename) is provided; if none are supplied it prints an error message and exits. It sets a flag named summary to True when more than one filename is passed, although that flag is not used further in the code. It initializes counters: tt (total trades/lines containing profit or loss) to 0, sl (stop‑loss or loss count) to 0, rs (running sum of profit/loss amounts) to 0, and day (a dictionary mapping date strings to aggregated amounts) to empty.

For each filename passed as an argument (skipping argv[0]), the program sets a formatting precision variable perc to 4 when the filename contains the substring "oanda", otherwise to 8. It reads the entire file contents through a helper module call JRRsupport.ReadFile(filename), strips leading/trailing whitespace, and splits the text into lines.

It then iterates through each line. Lines that are empty or that do not contain either the substring "Prft" or "Loss" are skipped. For lines that are processed, the code splits the line on spaces into a list named data, and then attempts to build a date key: it takes data[0], splits it on spaces (though data[0] will already be the first field), then splits that on '-' and constructs a date string using the first two components joined by a hyphen (year-month). Next it extracts a numeric substring pdata by splitting the original line on '/' and taking the second piece (index 1), stripping it, and then taking the first item before a comma. If the line contains "Prft" it converts pdata to a positive float and assigns it to pnl; if the line contains "Loss" it converts pdata to a float and negates it and assigns to pnl. That pnl value is added to the running sum rs. The code then adds pnl into the day dictionary under the date key, creating the entry if absent by initializing or incrementing the sum for that date.

The program increments tt by 1 for any line that contains either "Prft" or "Loss". It increments sl by 1 for any line that contains "Loss". After processing all lines in the file, it iterates over the sorted date keys in day and prints a line for each date. For each date it derives two labels from the input filename by splitting the filename on '.' and using parts at indices 2 and 3 (exac[2] and exac[3]) left‑justified into fixed widths; it prints the date, those two filename fields, and the aggregated day total formatted and rounded to perc decimal places.

After processing all files, the program prints a blank line. If there is at least one date key in day, it computes trades per month tpm as the integer division of total trade count tt by the number of distinct dates (len(day.keys())). It computes atm as the rounded average per trade rs/tt with perc decimal places, and apm as atm times tpm, rounded to perc decimals. Finally it prints lines showing: total profits/stoplosses/trades as rs (rounded to perc decimals) / sl / tt; the average per trade; average trades per month (tpm); and average profit per month (apm), each formatted to the precision determined earlier.
