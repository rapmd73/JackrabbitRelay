Section 1 - Non-Technical Description

This program reads a sequence of time-stamped price and volume records for a cryptocurrency pair, processes those records one at a time to compute trend strength and a simple directional comparison, and prints a rolling line of output for each record that includes the input fields plus the calculated indicators and a short crossing signal.

Section 2 - Technical Analysis

The script sets up a technical-analysis processing pipeline by importing a local module JRRtechnical as jrTA and creating an instance of jrTA.TechnicalAnalysis with parameters 'kraken', 'MAIN', 'ADA/USD', '1m', and 197. It then reads OHLCV data from the file 'ADAUSD.txt' via ta.ReadOHLCV, storing the returned list (or iterable) in ohlcv.

It defines numeric constants for indicator calculation: Period is set to 14, and index variables for fields in each OHLCV record are set such that Opening = 1, HighIDX = 2, LowIDX = 3, Closing = 4, Volume = 5. It also reserves column indices diP = 6 and diM = 7 to represent +DI and -DI indicator output locations.

The main processing loop iterates over each element named slice in the ohlcv collection. For each slice the program calls ta.Rolling(slice), which advances the internal rolling window or internal state of the TechnicalAnalysis object with the new record. After updating the internal state, it calls ta.ADX(HighIDX, LowIDX, Closing, Period). This instructs the TechnicalAnalysis instance to compute the Average Directional Index (ADX) and related directional movement indicators using the values at the specified field indices (high, low, close) and the given period (14). Next, the code calls ta.Cross(diP, diM), which computes a crossing relationship between the two internal indicator columns indexed by diP and diM; according to the surrounding comments, this yields a crossing distance and a crossing direction value where a positive value indicates +DI crossing over -DI (bullish), negative indicates bearish, and zero indicates no crossing. Finally, the script calls ta.Display(-1) to output a formatted line for the current rolling state; the example output in the file header shows that each display line contains date/time, open, high, low, close, volume, and then the computed +DI, -DI, ADX, crossing distance, and crossing indicator columns (when available), with earlier rows showing placeholders until sufficient data exists.

The script's entry point runs main() when executed as a program. The overall observable behavior is: read OHLCV rows from a file, feed them one-by-one to a TechnicalAnalysis object, compute ADX and a cross test between the +DI and -DI columns for each new row, and print a display line per row that combines the original OHLCV fields with the calculated indicators.
