# Section 1 - Non-Technical Description

This program continuously watches a set of active financial trades for a chosen account and asset, and when a trade reaches certain price or profit conditions it automatically sends a sell order to close that trade, then records a short log entry describing whether closing the trade resulted in a profit or a loss and how long the trade was open.

# Section 2 - Technical Analysis

The program initializes a JackrabbitRelay object and requires three command-line arguments to identify an exchange, an account, and an asset; if those are not provided it prints an error and exits. It determines the tick size ("onePip") for the chosen asset from relay.Markets and sets a target offset tp equal to ten pips.

The main loop runs forever. On each iteration it records the start time and retrieves the list of open trades for the selected asset via relay.GetOpenTrades; if that call fails it prints the exception and immediately continues to the next loop cycle. For each open trade it retrieves a current market ticker for the asset via relay.GetTicker; if that call fails it prints the exception and moves on to the next trade.

For each trade object o it parses the trade open time string (o['openTime']) to a datetime (dt) and computes a timestamp epoch. It reads the trade's current units (iu), the trade's entry price (price), and unrealized profit/loss (upl). It sets a side label 'L' for nonnegative units and 'S' for negative units. For long trades (iu > 0) the code checks if the current bid price from the ticker is greater than entry price + tp; if so it calls ProcessOrder to force a sell. It also prints a summary line for trades with positive unrealized profit. For short trades (iu <= 0) the code checks if the current ask price is less than entry price - tp; if so it calls ProcessOrder to force a sell. It also prints a similar summary line for shorts with positive unrealized profit. After processing all open trades it prints how long scanning that batch took.

ProcessOrder builds a new order dictionary representing a forced sell: it sets fields such as BruteSeller='Forced', Exchange='oanda', Account, Asset, Action='sell', EnforceFIFO='Yes', Price (the provided strike price), Units, Ticket (the original trade id), and OrderType='market'; it also assigns an Identity from relay.Active['Identity']. It then sends this order to the relay via relay.SendWebhook(newOrder). It retrieves an order identifier via relay.GetOrderID(result); if an order id is returned it fetches order details via relay.GetOrderDetails(OrderID=oid).

From the returned orderDetail array it extracts the close time from orderDetail[0]['time'] (trimming the fractional seconds to six digits and appending 'Z'), parses that into a datetime de and computes the trade duration as de - ds where ds is the datetime passed into ProcessOrder (the original open time). It reads the final profit/loss reported by the broker from orderDetail[-1]['pl'] as rpl and the executed price from orderDetail[-1]['price'] as sprice. It composes a log message indicating the new order id mapping to the original trade id, whether the result was a profit or loss, the direction label (derived from the sign of units), units, the original price, the executed price, the absolute value of reported profit/loss, and the duration. It writes this message to relay.JRLog.Write. If an exception occurs anywhere in ProcessOrder it writes a message to relay.JRLog.Write containing the exception line number and text.

Throughout the program, the printed output and log messages use the parsed and formatted values (prices formatted to a fixed number of decimal places, durations shown as Python timedelta strings). The loop repeats indefinitely, continuously scanning and acting on open trades.
