# Section 1 - Non-Technical Description

This program reads a stored list of currently tracked orders, selects those that match a specified market, account, and asset, then checks current market prices to report whether each tracked order would be in profit or show an unrealized loss; it also finds which tracked order has the highest and lowest recorded price and prints those along with elapsed time measurements for the checks.

# Section 2 - Technical Analysis

The script begins by importing modules, adding a specific path to sys.path, and defining filesystem paths including a data directory and a file called OliverTwist.Storehouse. It creates a locker object named OliverTwistLock using JRRsupport.Locker to coordinate access to the storehouse file.

A function ReadStorehouse(exchange, account, asset) is defined to return a list of orders from the storehouse that match the provided exchange, account, and asset. ReadStorehouse acquires the OliverTwistLock by repeatedly calling OliverTwistLock.Lock() until it returns the string 'locked', sleeping briefly between attempts. If the storehouse file exists, the file is read into buffer, stripped, split by newline into entries, and each non-empty entry is processed. Each entry is parsed using json.loads repeatedly while the entry remains a string to obtain a dictionary (the code attempts to handle nested JSON strings). If parsing fails for an entry, a logging call JRLog.Write is invoked with the broken line and processing of that entry is skipped. For parsed entries, if there are 'Order' or 'Detail' fields that are strings, those fields are parsed from JSON as well and their 'Identity' keys are removed if present. Entries whose Order.Exchange, Order.Account, and Order.Asset exactly match the function arguments are appended to OrphanList. After processing, the locker is released and OrphanList is returned.

In the main driver, a JackrabbitRelay object is created and command-line-like getters are used to obtain exchange, account, and asset; if fewer than four arguments are present the script prints a message and exits. ReadStorehouse is called with the exchange, account, and asset to obtain OpenOrders.

The script records the current time as StartTime and iterates over OpenOrders. For each order object o, it checks whether the 'Price' key exists; if missing it prints a message using the order's ID and continues to the next order. For orders with a Price, it requests the current market ticker from relay.GetTicker(symbol=asset). It compares ticker['Ask'] to the order Price minus 0.0010. If ticker['Ask'] is less than order Price - 0.0010, it calls Print(f'{o['ID']} in profit'); otherwise it prints the order ID and the absolute difference between the order Price and ticker Ask formatted to five decimal places followed by "unrealized loss". After iterating all orders it prints an empty line, computes and prints the elapsed time for the full sweep as EndTime - StartTime.

Next, the script initializes timing and variables to track the highest and lowest order prices seen: hPrice as negative infinity, lPrice as positive infinity, and hTrade and lTrade as None. It loops over OpenOrders again, skipping orders that lack a 'Price' key (printing the same "NO price data" message). For each order with a price it converts the stored price to a float, updates lTrade and lPrice if the price is lower than the current lPrice, and updates hTrade and hPrice if the price is higher than the current hPrice. After that loop it fetches the ticker again and compares ticker['Ask'] to lPrice - 0.0010; if ticker['Ask'] is less than that threshold it calls Print(f'{o['ID']} in profit'), otherwise it prints a message showing the absolute difference between lPrice and ticker['Ask'] formatted to five decimal places and "unrealized loss". It records the end time for the high/low sweep.

Finally, the script prints the highest and lowest trades by accessing hTrade['ID'] and hTrade['Price'], and lTrade['ID'] and lTrade['Price'], formatting the prices as floats with five decimal places. It prints the elapsed time for the high/low sweep as end - start. Throughout, the script uses a mix of print and Print calls and relies on the data read from the storehouse, the locker to serialize access, and ticker values returned by the relay object.
