# Section 1 - Non-Technical Description

This program queries a trading service for a list of market pairs, launches parallel worker tasks to fetch current price information for each market pair, waits for all workers to finish, then reads the results and prints a formatted summary of each market's pricing and related parameters for the account and exchange that were selected when the program started.

# Section 2 - Technical Analysis

The script begins by importing modules and adding a specific library path to sys.path. It constructs a SignalInterceptor instance from the JRRsupport module and defines a small helper GetRatio(decimal) that returns a string of the form "N:1" where N is 1/decimal converted to an integer.

A Help(args,argslen) function prints an instruction about requiring an exchange and account and then exits; this function is provided as a usage callback for the relay constructor.

GetChildTicker is the worker function launched for each market pair. It instantiates a JackrabbitRelay object using the provided exchange, account and asset (symbol) arguments. It calls relay.GetTicker(symbol) to obtain ticker data. If both ticker['Bid'] and ticker['Ask'] are zero, the function returns early (skipping that pair). It calls relay.GetMinimum(symbol) to obtain minimum and mincost, then stores those two values into the ticker dictionary under 'minimum' and 'mincost'. The function constructs two identifiers: one id string for the market (exchange.account.asset) and one memory id by appending ".memory". It creates two Locker objects from JRRsupport with those identifiers and a Timeout of 10. The function then locks the first Locker (mlock.Lock()), serializes the ticker with json.dumps, replaces certain characters in that serialized string, and stores the resulting string into the memory Locker with a 86400-second TTL using mdata.Put(86400,md). Finally it unlocks mlock. Any exception raised inside the try block is silently ignored (except block is pass).

The main() function constructs a JackrabbitRelay with Usage=Help. It obtains the selected exchange and account via relay.GetExchange() and relay.GetAccount(), and reads relay.Markets into markets. It parses additional optional command-line-like arguments via relay.GetArgsLen() and relay.GetArgs(): a search token srch (default 'ALL'), and optional numeric lower/upper filters srchLT and srchGT with associated floats sLT and sGT. Those parsed values are stored but not used except for filtering by name.

The program then iterates over relay.Markets (each key is a market pair name). For frameworks 'mimic' and 'ccxt' it skips pairs that start with '.' or contain '.d' or contain a space; it also skips markets where relay.Markets[pair] contains 'active' set to False. If a search string other than 'ALL' was provided, markets whose pair string does not contain that search substring are skipped. For each remaining pair, it calls interceptor.StartProcess(GetChildTicker,args=[exchangeName,account,pair]) to launch a child process or worker running GetChildTicker for that pair.

After starting a child, the main loop enforces a concurrency limit: while interceptor.GetChildren() is greater than the number of CPU cores os.cpu_count(), it sleeps 1 second at a time until the child count drops to or below the CPU count. After initiating processes for all eligible markets, main waits in a loop until interceptor.GetChildren() returns 0 (no children remain), sleeping 1 second between checks.

Next, main iterates over relay.Markets again, applying the same search filter as before. For each pair, it rebuilds the id and mid (memory id) strings, creates Locker objects for them (mlock and mdata), locks mlock, calls mdata.Get() and passes the result to json.loads, then erases the memory entry with mdata.Erase(), and unlocks mlock. It assigns that json.loads(mdata.Get()) result to data. It then attempts to parse a nested JSON payload by doing ticker=json.loads(data['DataStore'].replace('\\'',''')), wrapping this in a try/except and continuing to the next pair on any error. In other words, it expects data['DataStore'] to contain a JSON string, performs a string replacement on it and loads it into ticker.

After obtaining ticker, the program formats and prints output depending on relay.Framework. If relay.Framework is 'mimic' or 'ccxt', it reads marketType from relay.Markets[pair]['type'], converts spot+margin flags to 'margin' when present, reads id=relay.Markets[pair]['id'], and reads minimum and mincost from ticker. It then prints a single formatted line containing the pair name, id, marketType, minimum, mincost, ticker['Bid'], ticker['Spread'], and ticker['Ask'] with specific column widths and numeric formatting. If relay.Framework is 'oanda', it computes a margin ratio string via GetRatio using relay.Markets[pair]['marginRate'], reads financing long/short rates, and prints a formatted line containing the pair, margin ratio, long rate, short rate, ticker['Bid'], ticker['Spread'], and ticker['Ask'] with different column widths. For any other framework value, it prints 'Unrecognized framework' and exits with status 0.

At script execution time (when __name__ == '__main__'), main() is called inside a try block. If a KeyboardInterrupt occurs, the program prints 'Terminated'. The script therefore orchestrates parallel retrieval of tickers into a temporary shared memory area (via Locker/Put), waits for all child tasks to finish, reads those stored tickers back, and prints a summary table of market pricing information according to the framework used.
